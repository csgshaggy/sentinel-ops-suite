# Project Roadmap

## Overview
This roadmap outlines the planned evolution of the SSRF Command Console.  
It is divided into near‑term, mid‑term, long‑term, and exploratory phases to guide development priorities and strategic direction.

---

# 1. Near‑Term (0–3 Months)

## Authentication & RBAC
- Add token‑based authentication  
- Implement role‑based access control  
- Roles: Operator, Analyst, Admin  

## MODE SDK Enhancements
- Base MODE classes  
- Decorators for lifecycle hooks  
- Validation helpers  
- Improved manifest schema  

## Dashboard Improvements
- Enhanced anomaly visualization  
- Artifact preview panel  
- Improved run summary layout  

## Snapshot & Diff Enhancements
- Side‑by‑side diff viewer  
- Artifact‑level diffing  
- Improved drift detection  

---

# 2. Mid‑Term (3–6 Months)

## Distributed Execution Engine
- Worker nodes  
- Task queue  
- Parallel MODE execution  
- Execution scheduling  

## Plugin Marketplace
- MODE registry  
- Versioning and compatibility metadata  
- Secure installation workflow  

## Advanced Anomaly Detection
- Clustering  
- Fingerprinting  
- Heuristic‑based detection  
- Pattern recognition  

## Storage Backend Options
- S3 support  
- Azure Blob support  
- Hybrid local/cloud mode  

---

# 3. Long‑Term (6–12 Months)

## Full Web‑Based Operator Console
- Real‑time monitoring  
- Visual diffing  
- Artifact explorer  
- Live logs panel  

## Multi‑Tenant Model
- Tenant isolation  
- Tenant‑scoped MODEs  
- Tenant‑scoped storage  

## MODE Execution Sandbox
- Resource limits  
- Network policies  
- Execution quotas  

## Automated Reporting Engine
- HTML/PDF export  
- Report templates  
- Summary and anomaly reports  

---

# 4. Future / Exploratory

## Kubernetes Operator
- Cluster‑wide execution  
- Auto‑scaling MODE workers  
- CRD‑based configuration  

## MODE Signing & Verification
- Cryptographic signatures  
- Trust chain for MODE authors  
- Secure distribution  

## AI‑Assisted Anomaly Explanation
- Pattern‑based summarization  
- Contextual anomaly grouping  
- Suggested next steps  

## Replay & Simulation Mode
- Replay past runs deterministically  
- Simulate network conditions  
- Training and demo mode  

---

# Conclusion
This roadmap provides a structured vision for the evolution of the SSRF Command Console.  
It balances stability, extensibility, security, and operator‑grade functionality to support long‑term growth and professional use.
