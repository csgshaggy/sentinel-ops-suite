# Operator Training Guide

## Overview
This guide provides a structured training curriculum for operators, analysts, and testers using the SSRF Command Console.  
It covers essential skills, workflows, hands‑on exercises, and evaluation checkpoints to ensure operators can confidently run MODEs, interpret results, and use the dashboard effectively.

This guide is intended for onboarding, internal training, and SOC enablement.

---

# 1. Operator Role Overview

Operators are responsible for:

- Executing MODEs safely and consistently  
- Interpreting anomalies and artifacts  
- Using the dashboard for analysis  
- Creating snapshots and diffs  
- Maintaining deterministic workflows  
- Reporting findings to analysts or engineers  

Operators do **not** modify MODEs or backend code.

---

# 2. Training Objectives

By the end of this training, operators will be able to:

- Run any MODE using CLI or API  
- Navigate and interpret run directories  
- Use the dashboard to analyze results  
- Identify SSRF indicators and anomalies  
- Perform snapshot and diff analysis  
- Follow operational best practices  
- Troubleshoot common issues  

---

# 3. Prerequisites

Operators should have:

- Basic command‑line familiarity  
- Understanding of HTTP requests and responses  
- Awareness of SSRF concepts (optional but helpful)  

No development experience required.

---

# 4. Core Concepts

## 4.1 MODE
A modular execution unit that performs a specific task (e.g., SSRF scanning).

## 4.2 Run Directory
A timestamped folder containing:

\`\`\`
input.json
output.json
anomalies.json
artifacts/
logs/
\`\`\`

## 4.3 Artifacts
Raw and processed data generated by a MODE.

## 4.4 Dashboard
A read‑only UI for exploring runs, anomalies, and artifacts.

## 4.5 Snapshots & Diffs
Tools for comparing runs and detecting drift.

---

# 5. Training Module 1 — Running MODEs

## 5.1 Listing MODEs

\`\`\`
console modes list
\`\`\`

## 5.2 Running a MODE

\`\`\`
console run ssrf_basic_scan --targets example.com
\`\`\`

## 5.3 Overriding Configuration

\`\`\`
console run ssrf_basic_scan --timeout 10 --retries 3
\`\`\`

## 5.4 Hands‑On Exercise

1. Run `ssrf_basic_scan` against two targets  
2. Override timeout to 10 seconds  
3. Locate the run directory  
4. Open `input.json` and confirm overrides  
5. Open `output.json` and review results  

---

# 6. Training Module 2 — Understanding Run Output

## 6.1 Run Directory Structure

\`\`\`
runs/<timestamp>_<mode_name>/
├── input.json
├── output.json
├── anomalies.json
├── artifacts/
└── logs/
\`\`\`

## 6.2 Key Files

- **input.json** — What was executed  
- **output.json** — Final structured output  
- **anomalies.json** — Detected anomalies  
- **artifacts/** — Raw and processed data  
- **logs/** — Execution logs  

## 6.3 Hands‑On Exercise

1. Open a run directory  
2. Identify anomalies  
3. Inspect raw artifacts  
4. Review logs for errors  

---

# 7. Training Module 3 — Dashboard Usage

## 7.1 Starting the Dashboard

\`\`\`
python -m dashboard
\`\`\`

## 7.2 Dashboard Features

- Run history  
- Artifact viewer  
- Anomaly viewer  
- Log viewer  
- Diff viewer  

## 7.3 Hands‑On Exercise

1. Open dashboard  
2. Select a run  
3. View anomalies  
4. Inspect artifacts  
5. Compare two runs  

---

# 8. Training Module 4 — Snapshot & Diff Analysis

## 8.1 Creating a Snapshot

\`\`\`
console snapshot create <run_id>
\`\`\`

## 8.2 Diffing Runs

\`\`\`
console diff <run1> <run2>
\`\`\`

## 8.3 Use Cases

- Detecting drift  
- Comparing environments  
- Regression analysis  
- Change validation  

## 8.4 Hands‑On Exercise

1. Run the same MODE twice  
2. Create a snapshot of the first run  
3. Diff the second run against the snapshot  
4. Identify differences  

---

# 9. Training Module 5 — Interpreting Anomalies

## 9.1 Common SSRF Indicators

- Access to internal metadata endpoints  
- Unexpected redirects  
- Internal IP leakage  
- Error messages revealing internal services  

## 9.2 Anomaly Severity Levels

- **Informational** — unusual but not harmful  
- **Suspicious** — requires review  
- **Critical** — strong SSRF indicator  

## 9.3 Hands‑On Exercise

1. Run a MODE against known SSRF targets  
2. Identify anomalies  
3. Categorize them by severity  
4. Document findings  

---

# 10. Training Module 6 — Troubleshooting

## 10.1 Common Issues

### MODE fails preflight
- Invalid targets  
- Invalid overrides  
- Missing config keys  

### Dashboard won’t load
- Port conflict  
- Missing dependencies  
- Service not running  

### MODE not listed
- Missing `mode.yaml`  
- Validation failure  

## 10.2 Hands‑On Exercise

1. Intentionally misconfigure a MODE  
2. Observe preflight failure  
3. Fix configuration  
4. Re‑run successfully  

---

# 11. Training Module 7 — Operational Best Practices

- Always validate MODEs before running  
- Never modify run directories  
- Use snapshots for long‑term comparisons  
- Document anomalies clearly  
- Use CLI for automation  
- Use dashboard for analysis  
- Follow security model guidelines
